## Wymagane funkcjonalności

- Demonstracyjny system rezerwacji: usługa → forma → kalendarz → dane → podsumowanie →
  płatność demo (BLIK/karta/przelew/Apple Pay/Google Pay) → potwierdzenie z numerem
  AN-RRRR-XXXX, plik .ics, zapis localStorage
- Formularze (kontakt, B2B, newsletter) z pełną walidacją i symulacją wysyłki
- Baner cookies z kategoriami (niezbędne zawsze aktywne, opcjonalne domyślnie wyłączone)
- Menu mobilne pełnoekranowe (ARIA, pułapka fokusu, scroll-lock)
- Stały mobilny przycisk „Umów spotkanie" (nie na /rezerwacja)

## Estetyka

Premium, spokojna, elegancka, ciepła; złamana biel + beż + szałwia + świerkowa zieleń +
akcent miedziany; Fraunces (display) + Albert Sans (tekst); dużo przestrzeni; subtelne
animacje respektujące `prefers-reduced-motion`; zero agresywnych efektów/popupów/odliczania.
Sygnatura: miedziana „nić rozmowy" (SVG) + kodowanie kolorem 3 ścieżek.

## Wymagania mobilne

Mobile-first; działa od 320 px; zero poziomego scrolla; pola dotykowe ≥ 44 px;
safe-area (notch); rezerwacja wygodna jedną ręką.
